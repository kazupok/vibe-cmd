## 🔄 タスク管理フロー

1. **タスク作成**: GitHub Issuesで管理、詳細仕様はissue本文に記載
2. **実装開始**: Issueをassignee設定、ステータス更新
3. **実装完了**: Issueをclose、`current.md` のチェックボックスを更新
4. **バージョンリリース**: 完了版を `versions/` に保存
